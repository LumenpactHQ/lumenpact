# Coding standards

## Rust

- keep contract logic explicit and easy to audit,
- prefer clear names over clever abstractions,
- preserve existing error handling patterns,
- and add tests whenever behavior changes.

## TypeScript and React

- keep components focused and composable,
- avoid hidden side effects in shared helpers,
- and prefer descriptive names for state and utility functions.

## Documentation

- update docs when public behavior changes,
- keep examples accurate,
- and avoid placeholder content.
