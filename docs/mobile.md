# Mobile application strategy

The repository currently focuses on a web-based experience. That makes sense for the initial release because it lowers the barrier to contribution and makes the contract workflow easier to review.

## Current status

- the primary user interface is the Next.js app,
- wallet interactions are web-compatible,
- and the contract layer is suitable for a future mobile client.

## Direction

A mobile app can be added later by reusing the same contract and account model. The experience should keep the same core concepts: create, review, resolve, and cancel commitments with clear wallet and network feedback.
